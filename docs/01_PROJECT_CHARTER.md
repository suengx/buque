# 项目章程：补雀 BuQue

## 1. 项目背景

跨境电商库存管理通常涉及多个平台、多个仓库、多个 SKU、多个责任角色以及较长的采购与物流提前期。

在人工 Excel 监控模式下，计划专员需要持续盯销量、库存、在途、预测、运营计划和采购交期。风险往往不是没有数据，而是数据分散、口径不统一、异常信号发现晚、解释依赖个人经验、建议动作缺少闭环。

补雀的项目目标，是将现有人工监控经验和 Excel 规则系统化，逐步建设为可配置、可解释、可追溯、可反馈的计划监控预警 Agent。

---

## 2. 项目使命

> 让跨境电商团队在库存风险变成损失之前，提前看见、解释并处理它。

---

## 3. 产品目标

补雀的长期目标包括：

- 统一 SKU 销量、库存、在途、预测和运营计划的监控口径
- 将计划团队现有 Excel 规则沉淀为系统化规则引擎
- 每日自动识别断货、滞销、销量异常、预测偏差和数据异常
- 对异常 SKU 输出主解释、次解释、关键证据和冲突处理原则
- 对不同风险输出建议动作、责任角色和处理时效
- 通过人工反馈沉淀样本，为二期预测建议和长期 Agent 化打基础

---

## 4. 目标用户

| 角色 | 核心诉求 |
|---|---|
| 计划专员 | 少盯表，更快找到今日高风险 SKU |
| 计划主管 | 快速掌握风险规模、优先级和责任闭环 |
| 运营主管 | 确认销量变化是否来自活动、广告、价格或链接变化 |
| 采购 / 物流 | 处理催交、延期、在途异常和补货可行性 |
| 管理层 | 看每日风险概览、趋势变化和系统化提效效果 |
| 开发 / 数据 | 有明确字段、规则、参数、输出和验收口径 |

---

## 5. 核心收益

| 收益类型 | 说明 |
|---|---|
| 效率收益 | 降低计划专员每日盯表与初筛耗时 |
| 风险收益 | 提前发现断货、滞销、预测偏差和数据异常 |
| 协同收益 | 将风险分发给正确责任角色，减少口头沟通损耗 |
| 沉淀收益 | 把 Excel 规则、解释经验和人工反馈沉淀为系统资产 |
| 预测收益 | 为二期预测修正建议提供样本与反馈基础 |

---

## 6. 范围定义

### 纳入范围

- SKU 基础资料
- 销量、库存、在途、预测、运营计划等监控数据
- DOS、销量波动、预测偏差等核心指标
- 断货、滞销、销量异常、预测偏差、数据异常风险
- 红、橙、黄、绿风险等级
- 日报、风险清单、单 SKU 分析卡
- 建议动作与责任角色
- 人工反馈记录

### 暂不纳入或需后置

- 自动下采购单
- 自动调拨
- 自动改价、清货或投放调整
- 自动修改正式预测
- 模型自行调整阈值
- 未经人工确认的高风险动作执行

---

## 7. 成功标准

补雀是否成功，不看“AI 说得是否像人”，而看以下事实：

- 系统结果能否与现有 Excel 监控逻辑对齐
- 红橙灯清单是否确实有处理价值
- 计划专员是否减少盯表和初筛时间
- 异常解释是否基于证据，而不是泛泛而谈
- 建议动作是否能指向责任角色和处理时效
- 人工反馈是否能稳定沉淀下来
- 二期是否具备预测建议的数据基础

---

## 8. 关键原则

### 先复制，再优化

一期先复制已验证的 Excel 逻辑，不急于一开始就重做预测模型或复杂 AI 判断。

### 先规则，再 Agent

规则引擎负责稳定判断，Agent 负责解释、建议和总结。Agent 不直接改底层规则。

### 先建议，再确认

红灯、预测修正、采购、调拨、去化等动作必须人工确认。

### 先闭环，再学习

没有人工反馈闭环，就不应谈学习型预测建议。

### 先可信，再自动化

先让业务相信结果可用，再逐步扩大范围和自动化程度。
